# PLAN

## Goal

Androidで文字起こしくんに音声ファイルをアップロードするとき、従来の音声選択を残しながら、Android Files/フォルダから直接音声ファイル1件を選べる入口を追加する。

## Current Findings

- `src/pages/UploadPage.tsx` のアップロード欄は単一の `input type="file"` で、`accept="audio/*,...` を指定している。
- Android Chrome/PWAでは `accept="audio/*"` により音声専用の選択UIに寄り、Files/ダウンロード等から探す入口が出にくい端末差があり得る。
- クライアントは選択された音声をブラウザ内で mono 16kHz MP3 に圧縮し、`/api/transcribe` へ送っている。
- サーバー側はフォームに入ってきた `File` を受け取り、OpenAIへ投げる前の明示的な非音声拒否は弱い。

## Decisions

- アップロード欄は2導線にする。
- `音声から選択` は従来通り `accept` 付きの入力を使う。
- `ファイルから選択` は `accept` なしの入力を使い、Android Files/ダウンロード等から探しやすくする。
- 選択後の共通バリデーションで、対応候補は `.mp3`, `.m4a`, `.mp4`, `.wav`, `.aac`, `.ogg`, `.webm`, `.opus` と `audio/*` MIME とする。
- `.amr`, `.3gp`, `.3gpp`, `.awb` は従来通り非対応として止める。
- サーバー側でも同等の最低限バリデーションを行い、非音声をOpenAI送信前に拒否する。
- 実装後はビルド、レビュー、本番デプロイまで行う。

## Open Questions Resolved

- Android向け導線: 2導線を採用する。
- ロールアウト範囲: 本番反映まで実施する。
- 実機確認: デプロイ後、本番URLでAndroid実機確認できる状態にする。

## Scope

- 対象: `src/pages/UploadPage.tsx`, `src/constants.ts`, `src/index.css`, `functions/_lib.ts`
- 非対象: フォルダ丸ごとのアップロード、録音機能の新規追加、文字起こし処理や保存形式の変更、D1/R2スキーマ変更

## Steps

1. クライアントの対応拡張子/MIME判定を共通化する。
2. アップロード画面に `音声から選択` と `ファイルから選択` の2ボタンを追加する。
3. `ファイルから選択` で非音声が選ばれた場合は、アップロード前に分かりやすく止める。
4. サーバー側のフォームファイル収集時にも同等の音声判定を加える。
5. ビルド、差分レビュー、必要な修正、本番デプロイを行う。

## Validation

- `npm run build`
- 実装差分レビューでP0/P1/P2が残らないことを確認する。
- 本番デプロイ後、Android実機で `ファイルから選択` から `.m4a` または `.mp3` を選べることを確認する。
- 従来の `音声から選択` が使えることを確認する。
- 非対応形式と非音声ファイルがアップロード前またはOpenAI送信前に止まることを確認する。

## Risks

- Androidのファイルピッカー表示は端末・Chrome/PWA実装に依存するため、このMac上のビルドだけでは最終表示を完全確認できない。
- `accept` なし導線では非音声も選べるため、クライアント/サーバー双方の判定で止める必要がある。
